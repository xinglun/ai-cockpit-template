# プロジェクト用語集 (Glossary)

このドキュメントは、AI Agent がコードベースを理解し、実装する際に従うべきドメイン用語やアーキテクチャ上の定義をまとめたものです。
AI Agent は作業を開始する前に必ずこのドキュメントを読み込み、定義に沿って命名や設計を行ってください。

## 1. ドメイン用語定義

*以下は Sentinel プロジェクトにおける用語の定義例です。プロジェクトに応じて書き換えてください。*

| 用語 (英語) | 用語 (日本語) | 定義 / 制約 |
| :--- | :--- | :--- |
| **Capital Absorption** | 资金吸収 / 资金调达 | 投资家から资金を吸い上げてプールする一连の业务プロセス。`FundingPool` や `CapitalAbsorptionService` などの命名规则に従うこと。 |
| **IPO Queue** | IPO待机キュー | 新规公开株の配分待ち状态にある申请の优先度付きキュー。`CapitalAbsorptionIpoQueueStore` で永続化される。 |
| **Sentinel** | センチネル / 监视ゲート | システム全体のデータ整合性とレートリミットを监视する中核モジュール。 |

## 2. アーキテクチャ境界

- **Infrastructure Layer**: 外部データベースやキューへのアクセスを担当するレイヤー。ドメインロジックをここに書かないこと。
- **Domain Layer**: 纯粋なビジネスルールを含むレイヤー。フレームワークやデータベースライブラリへの直接の依存を禁止する。
- **Presentation Layer**: HTTP / API エントリーポイント。バリデーション以外のビジネスロジックはドメインサービスに委譲すること。
